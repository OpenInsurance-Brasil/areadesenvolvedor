    API de status

    Exemplo de código

    GET https://api.seguros.com.br/open-insurance/discovery/v1/status HTTP/1.1
Host: api.seguros.com.br
Accept: application/json


    var req = new XMLHttpRequest();
req.setRequestHeader("Accept", "application/json");
req.open("GET", "https://api.seguros.com.br/open-insurance/discovery/v1/status", true);
req.send();

    API de outages

    Exemplo de código

    GET https://api.seguros.com.br/open-insurance/discovery/v1/outages HTTP/1.1
Host: api.seguros.com.br
Accept: application/json


    var req = new XMLHttpRequest();
req.setRequestHeader("Accept", "application/json");
req.open("GET", "https://api.seguros.com.br/open-insurance/discovery/v1/outages", true);
req.send();

    Na estrutura de retorno exemplificada abaixo, no caso em que o parâmetro isPartial devolvido seja true, o array unavailableEndpoints deve conter a lista de endpoints indisponíveis:

    {
  "data": {
    "outages": [
      {
        "outageTime": "2020-07-21T08:30:00Z",
        "duration": "PT2H30M",
        "isPartial": false,
        "explanation": "Atualização do API Gateway",
        "unavailableEndpoints": [
          "https://api.seguros.com.br/open-insurance/discovery/v1/outages"
        ]
      }
    ]
  },
  "links": {
    "self": "https://api.seguros.com.br/open-insurance/discovery/v1/outages"
  },
  "meta": {
    "totalRecords": 1,
    "totalPages": 1
  }
}

    GET /channels/<versão>/branches

    Versão
    1

    Visão geral

    Obtém a lista de dependências próprias da instituição de seguros.

    Dependência própria é o espaço físico destinado ao atendimento ao público.

    Tags: Agência (Branch), CNPJ (CNPJ Number), Marca (Brand), Dependência (Branch), Instituição de Seguros (Company), Posto de Atendimento Bancário - PAB (Branch), Posto de Atendimento Eletrônico – PAE (Branch) e Unidade Administrativa Desmembrada – UAD (Branch).
